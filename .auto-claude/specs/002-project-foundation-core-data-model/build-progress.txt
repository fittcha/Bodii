This subtask was already completed in a previous session. The implementation provides essential type conversion methods between Core Data's Decimal type and Swift's native Double/Int types.

**Why Decimal in Core Data?**
- Prevents floating-point precision errors in health/financial data
- Ensures accurate calculations for body measurements (weight, body fat %, etc.)
- Example: 67.5kg stored as exact Decimal, not approximate binary floating-point

**Key Features:**
- Bidirectional conversion: Decimal ↔ Double/Int
- Rounding with configurable decimal places
- UI formatting without thousand separators
- Comprehensive Korean documentation explaining use cases
